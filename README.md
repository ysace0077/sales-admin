# 통합 매출 관리 시스템

## 📋 프로젝트 개요

기존의 세 가지 프로젝트(sale_report_viewer, customer_viewer, sales_calendar)를 하나로 통합한 효율적인 매출 관리 솔루션입니다.

## ✨ 주요 특징

### 1. 단일 데이터베이스 공유
- 하나의 SQLite 파일로 모든 기능 사용
- 데이터 중복 없이 효율적인 메모리 관리
- 파일 한 번 로드로 모든 탭에서 즉시 사용 가능

### 2. 3가지 통합 기능

#### 📈 매출 리포트
- ACE/ESSA 브랜드별 매출 분석
- 년도/월/판매자별 필터링
- 실시간 통계 (총 매출, 거래 건수, 판매 수량, 평균 거래액)
- 월별 매출 추이 차트
- 판매자별 매출 비교 차트
- Excel 내보내기 기능

#### 👥 고객 조회
- ACE/ESSA 브랜드별 고객 데이터 조회
- 년도/월/고객명/지역별 검색
- 카드 형태의 직관적인 결과 표시
- 페이지네이션 (50건씩)
- Excel 내보내기 기능

#### 📅 매출 캘린더
- 월별 캘린더 형태로 일일 매출 시각화
- 브랜드/판매자별 필터링
- 일간 평균 및 월간 매출 통계
- 주말/평일 구분 표시
- 직관적인 색상 코딩 (흑자/적자)

## 🚀 사용 방법

### 1. 파일 준비
- SQLite 데이터베이스 파일 준비 (sdata.db 또는 sales_data.db)
- 필수 테이블: `data_(ACE)`, `data_(ESSA)`

### 2. 실행
1. `index.html` 파일을 웹 브라우저로 열기
2. SQLite 파일을 드래그 앤 드롭 또는 클릭하여 선택
3. 데이터 로드 완료 후 원하는 탭 선택

### 3. 탭 전환
- **매출 리포트**: 전체적인 매출 분석 및 차트
- **고객 조회**: 개별 고객 거래 내역 검색
- **매출 캘린더**: 일별 매출 현황 캘린더 뷰

## 📊 데이터베이스 구조

### 필수 테이블
- `data_(ACE)`: ACE 브랜드 판매 데이터
- `data_(ESSA)`: ESSA 브랜드 판매 데이터

### 필수 컬럼
- `YEAR`: 년도
- `MONTH`: 월
- `판매일`: 판매 날짜
- `고객명`: 고객 이름
- `고객번호`: 고객 번호
- `상품명`: 상품 이름
- `할인가`: 판매 금액
- `수량`: 판매 수량
- `판매자`: 판매자 이름
- `지역1`, `지역2`, `지역3`: 지역 정보

## 🎨 기술 스택

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Database**: SQLite (sql.js)
- **Charts**: Chart.js
- **Export**: SheetJS (xlsx)

## 💡 장점

### 효율성
- **단일 파일 로드**: 데이터베이스 파일을 한 번만 로드하여 모든 기능 사용
- **메모리 최적화**: 공유 데이터베이스 인스턴스로 메모리 사용량 최소화
- **빠른 전환**: 탭 전환 시 데이터 재로드 불필요

### 사용성
- **통합 인터페이스**: 하나의 화면에서 모든 기능 접근
- **일관된 UX**: 모든 탭에서 동일한 디자인 언어
- **반응형 디자인**: 모바일/태블릿/데스크톱 모두 지원

### 유지보수
- **모듈화된 구조**: 각 기능이 독립적인 모듈로 분리
- **확장 가능**: 새로운 탭/기능 추가 용이
- **코드 재사용**: 공통 기능 중복 제거
